  <action>
    Create `kbd_backlight/hardware/backlight.py` implementing the `BacklightController` class. Follow the research pattern exactly — the write format and glob are confirmed working on this hardware.

    **Module-level constants:**
    ```python
    SYSFS_GLOB = "/sys/class/leds/asus::kbd_backlight*/kbd_rgb_mode"

    MODES = {
        "static":      0,
        "breathing":   1,
        "color_cycle": 2,
        "strobe":      3,
    }
    ```
    Note: mode 3 (strobe) may be silently ignored by this specific hardware unit (research open question). Include it — do NOT remove it. Hardware testing in Phase 1 verification will determine if it works.

    **Exception class:**
    ```python
    class HardwareNotFoundError(RuntimeError):
        """Raised when the ASUS keyboard backlight sysfs path cannot be found."""
    ```

    **BacklightController class:**
    - `__init__(self, sysfs_path: str | None = None)`: If `sysfs_path` is None, call `self._discover()`. Store as `self._path = Path(sysfs_path)`.
    - `_discover() -> str` (static method): Glob `SYSFS_GLOB`, take `matches[0]` if any exist, else raise `HardwareNotFoundError` with this exact message: `"Could not find ASUS keyboard backlight device at /sys/class/leds/asus::kbd_backlight*/kbd_rgb_mode. Is the asus-nb-wmi kernel module loaded?"`
    - `apply(self, mode: str, r: int, g: int, b: int, speed: int, persist: bool = False) -> None`:
      - Validate mode: if not in MODES, raise `ValueError(f"Unknown mode '{mode}'. Valid modes: {list(MODES)}")`
      - Validate RGB: if not all `0 <= c <= 255` for c in (r, g, b), raise `ValueError(f"RGB values must be 0–255, got ({r}, {g}, {b})")`
      - Validate speed: if speed not in (0, 1, 2), raise `ValueError(f"Speed must be 0, 1, or 2, got {speed}")`
      - Build payload: `f"{1 if persist else 0} {MODES[mode]} {r} {g} {b} {speed}\n"`
      - Write: `self._path.write_text(payload)`
      - Wrap `PermissionError`: catch it, re-raise with message: `f"Cannot write to {self._path}. Is the udev rule installed? Run: sudo install/setup-permissions.sh"`
    - `@property path(self) -> Path`: returns `self._path` (for testing/debugging)

    **Imports needed:** `from pathlib import Path`

    Do NOT use subprocess, os.system, or any shell invocation. Direct `Path.write_text()` only.
    Do NOT catch any exception other than `PermissionError` in `apply()` — let `IOError` and others propagate naturally so bugs are visible.
    Do NOT default `persist=True` anywhere.
  </action>
