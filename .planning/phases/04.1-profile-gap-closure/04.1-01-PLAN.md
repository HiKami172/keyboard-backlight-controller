<objective>
Move `_restore_last_profile()` in `application.py` from inside the `if not self._tray_only:` branch to just before it, so the last-used profile is applied to the keyboard hardware on every first activation — including `--tray-only` autostart launches.

Purpose: Close PROF-04 gap identified in v1.0 milestone audit. Currently, launching via XDG autostart (--tray-only) leaves the keyboard at firmware default instead of restoring the user's configured profile.
Output: `kbd_backlight/ui/application.py` with the one-line placement fix applied.
</objective>

  <files>kbd_backlight/ui/application.py</files>
  <action>
In `_on_activate()`, the current lines 74-76 read:

```python
if not self._tray_only:
    self._window.present()
    self._restore_last_profile()
```

Change them to:

```python
self._restore_last_profile()   # PROF-04: always restore on first activation
if not self._tray_only:
    self._window.present()
```

IMPORTANT constraints:
- Do NOT move the call before `self._controller` is initialized (lines 43-58) or before `self._window` is created (lines 60-67). The correct placement is after both are initialized, immediately before the `if not self._tray_only:` block — exactly as shown.
- Do NOT change the `_restore_last_profile()` method itself. It already uses `persist=False` (cmd=0) and wraps the apply() call in `try/except Exception: pass`. Both of these are correct — leave them unchanged.
- Do NOT add a second call inside the `if not self._tray_only:` branch. One call before the conditional is sufficient for both code paths.
- The comment "PROF-04: always restore on first activation" makes the intent clear for future readers.
