## Decisions Made

- `_do_rename` does NOT call `dialog.close()` inside `except ValueError` — the dialog must stay open so the user can correct the name without re-opening and re-typing. This is the critical anti-pattern avoided per RESEARCH.md Pitfall guidance.
- `entry.set_text(old_name)` is mandatory in `_show_rename_dialog` — pre-populating forces the user to only edit the diff, not retype from scratch.
- Rename row placed between Save and Delete, following ascending destructiveness order.
- Guard condition (`if not names or idx >= len(names): return`) mirrors the existing guard in `_on_delete_clicked` and `_on_profile_selected` for consistent placeholder protection.
